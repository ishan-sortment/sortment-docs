# Profiles

The **Profiles** section in Sortment is where you view and manage individual user data. Each profile is defined against the User ID as selected during schema setup. Use the **search bar** at the top to find profiles by **UserID**.

<figure><img src="../../.gitbook/assets/image (18).png" alt=""><figcaption></figcaption></figure>

By default, no column is selected. You can add attributes like name, email, and sign-up date to the view.

By default, Sortment shows you any dynamic traits setup for your profiles. Also, event history and subscription preferences is available.

#### **Purpose**

* Understand user behavior and attributes.
* Monitor engagement and event activity.
* Manage channel subscription preferences (Ex: Email, SMS, WhatsApp).
